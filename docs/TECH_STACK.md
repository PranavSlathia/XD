## Worker runtime conventions

| Concern | Choice | Notes |
|---|---|---|
| Concurrency model | **Single-process asyncio** + `asyncio.Semaphore` per external API + `aiolimiter` token-bucket | CPU-bound work (sentence-transformers, HTML parse) offloaded via `anyio.to_thread.run_sync` |
| Graceful shutdown | **SIGTERM handler that drains in-flight** then exits within 10s | Docker `compose down` semantics; no half-done Postgres / Claude calls |
| Worker entry-points | **One entry-point per worker** in `dh.workers.<name>:main` | containers: `dh-worker-a2`, `dh-worker-rdap`, `dh-worker-wayback`, `dh-worker-classifier`, `dh-scheduler`, `dh-api` |
| IPC | **Postgres as source of truth** + **Redis pub/sub** for SSE dashboard notifications | no direct worker-to-worker HTTP; workers poll/claim work via Postgres rows |
